# Lab 04 - MVP Feature List

## MVP Decision Rule

The MVP includes the minimum set of features required to demonstrate that students can identify campus buildings, access campus information, and navigate the campus efficiently using AI image recognition and a chatbot.

## MoSCoW Prioritization

| Feature ID | Feature Name | Problem Solved | Related User Story | Priority (Must/Should/Could/Won't) | Technical Complexity (1-5) | User Value (1-5) | Evidence Strength (1-5) | Include in Final Prototype? |
|---|---|---|---|---|---:|---:|---:|---|
| F01 | Upload Building Photo | Students cannot identify campus buildings. | US-01 | Must | 4 | 5 | 5 | Yes |
| F02 | AI Building Recognition | Students do not know which building they are looking at. | US-01 | Must | 5 | 5 | 5 | Yes |
| F03 | Chatbot Question & Answer | Students cannot quickly find campus information. | US-02 | Must | 4 | 5 | 5 | Yes |
| F04 | Building Information Page | Campus information is scattered across different sources. | US-03 | Must | 2 | 5 | 5 | Yes |

| F06 | Campus Map Navigation | Students get lost when visiting unfamiliar buildings. | US-05 | Should | 3 | 4 | 4 | Yes |

---

## Must-Have Features

- Upload building image
- AI building recognition
- Chatbot for campus information
- Building details (facilities, clubs, hours, floor map)
- Building search
- Admin management page
- Input validation
- Confirmation message
- Campus map with navigation
- Feedback and rating system

---

## Should-Have Features

- Campus navigation
- Campus map browsing
- Dashboard summary for administrators
- Multi-language support
- Search suggestions with auto-complete

---

## Could-Have / Future Features

- Voice search
- Indoor navigation
- Student login system
- Push notifications
- Multi-language chatbot
- Personalized recommendations
- Voice chatbot interaction
- Building occupancy status

---

## Not in MVP

The following features are intentionally excluded from the MVP because they require more advanced technologies or are outside the scope of the semester project:

- Voice recognition and voice commands
- Indoor GPS navigation
- Real-time user location tracking
- Student authentication with university accounts
- Push notifications for campus events
- AI recommendation system
- Mobile application version

These features can be considered for future development after the core prototype has been successfully implemented and validated.
