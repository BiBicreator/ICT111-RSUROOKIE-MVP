# Business Model Canvas

## Project Title

RSU Campus Buddy (Smart Campus Chatbot for Student Services with Image Recognition)

## 1. Customer Segments

**Primary Users**
- Freshmen at Rangsit University
- International students

**Secondary Users**
- Current RSU students
- Campus visitors
- University staff

---

## 2. Value Proposition

RSU Campus Buddy helps students quickly identify campus buildings and access accurate information about facilities, services, operating hours, student clubs, and university regulations. By combining Google Teachable Machine image recognition, a searchable campus database, and a chatbot, the system provides a faster and more convenient way to find campus information from a single platform.

---

## 3. Channels

Users can access the system through:

- Web application
- Mobile web browser
- QR codes placed around campus
- University orientation sessions
- Student social media groups

---

## 4. Customer Relationships

- Self-service web application
- AI-assisted chatbot for campus questions
- Administrator-managed campus information
- Continuous updates based on user feedback

---

## 5. Revenue Streams / Sustainability Logic

This project is developed as an educational MVP and does not generate direct revenue.

Its long-term value comes from:

- Improving the campus experience for students
- Reducing repetitive questions to university staff
- Providing a centralized source of campus information
- Supporting university digital transformation

If adopted in the future, the university could maintain and expand the system as an official campus service.

---

## 6. Key Resources

- Flask web application
- HTML, CSS, and JavaScript
- Google Teachable Machine image recognition model
- Custom campus information database
- Chatbot
- Admin management system
- GitHub repository

---

## 7. Key Activities

- Collect campus building information
- Train the Google Teachable Machine model
- Develop the web application
- Design the user interface
- Manage and update the campus database
- Test the system with students
- Maintain project documentation

---

## 8. Key Partners

- Rangsit University
- Course instructor
- Project team members
- Student volunteers for testing
- Student clubs and campus organizations
- Google Teachable Machine platform

---

## 9. Cost Structure

- Development time
- Image collection and preparation
- Website hosting (if deployed)
- Domain name (optional)
- System maintenance
- Database maintenance
- Internet and cloud services

---

## Final Note

The Business Model Canvas supports the MVP by focusing on the core features that provide the greatest value to students. The image recognition system, building search, chatbot, and administrator-managed database directly address the validated problems identified during customer discovery. These components will guide the implementation of the final prototype while remaining achievable within the semester timeline.
