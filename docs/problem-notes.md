### Problems or Difficulties

1. Most team members were using GitHub for the first time and needed time to learn how repositories, commits, and issues work.

2. The team initially had difficulty understanding the required repository structure and how to organize project documents correctly.

3. Some members were unfamiliar with Markdown (.md) files and needed guidance on creating and formatting documentation.

4. It took time for the team to decide on a project idea and assign roles based on each member's strengths and interests.

5. Coordinating schedules for meetings and collaboration was challenging due to different class and personal schedules.
