Wireframe Specification

## Required Wireframe Screens

| Wireframe File | Purpose | Minimum Elements to Show | Related Requirements |
|---|---|---|---|
| /wireframes/homepage.png | Entry point of the system | Logo, navigation, search bar, upload image button, chatbot button | FR-01, FR-02 |
| /wireframes/input-form.png | Upload building image | Image upload, preview, identify button, prediction result | FR-03, FR-04 |
| /wireframes/record-list.png | Browse campus buildings | Search bar, categories, building list/cards | FR-05, FR-06 |
| /wireframes/detail-view.png | Display building information | Building photo, services, facilities, operating hours, clubs, regulations | FR-07, FR-08 |
| /wireframes/chatbot.png | Campus chatbot | Chat window, user input, chatbot responses, suggested questions | FR-09, FR-10 |
| /wireframes/admin-view.png | Manage building information | Add, edit, delete buildings, upload images, update information | FR-11, FR-12 |

## Wireframe Quality Rules

- Use consistent navigation and labels across all screens.
- Show realistic RSU building names and sample information.
- Every screen must support one or more system requirements.
- Wireframes may be created using Figma, Canva, PowerPoint, or HTML/CSS.
